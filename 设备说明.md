## 1. 概述

### 1.1 架构说明

本集群是一套**零信任（Zero Trust）+ 纯 IPv6 直连**的异构 AI 计算平台，覆盖云端、家庭内网与跨地域接入三大场景：

- **公网入口（IPv4）**：阿里云 ECS（`cloud`，43.119.32.180）作为唯一对外暴露的公网节点，运行 OpenResty 反向代理 + 统一认证网关（aigc-auth）。所有对外服务均先经 `auth_request` 校验 Cookie/Session，未登录请求被重定向至登录页。cloud 因阿里云无公网 IPv6，仅走 IPv4。
- **家庭内网入口（IPv6）**：`workstation`（Ubuntu，4× RTX PRO 6000 Blackwell）通过电信 IPv6 公网直连，运行 Caddy 网关，使用 `forward_auth` 委托 `auth.dgmt.top` 做认证校验，与云端形成双活入口。
- **IPv6 直连**：所有家庭设备（workstation / studio01-04 / spark01-02 / openclaw01-04 / pc01 / nas）均使用基于 MAC 的稳定 IPv6 地址（`240e:b8f:1d22:e700::/64`）直连，无需 Tailscale 中转。
- **SSH 跳板**：cloud 上运行 11 个 `ssh-fwd-2200X` systemd 服务，将 `lab.wineryz.top:22002~22012` 转发至各内网设备的 22 端口，提供全球可达的运维通道（IPv4 备用）。
- **DDNS**：workstation 上 `ddns-aliyun.timer` 每分钟刷新 `*.dgmt.top` 的 AAAA 记录，使家庭 IPv6 前缀变化后仍可直达。

### 6.3 公网访问路径

1. **IPv4 用户** → `*.dgmt.top` A 记录 → cloud (43.119.32.180) → OpenResty → `auth_request` 校验 → 后端服务
2. **IPv6 用户**（aigc/erp/toiv）→ `*.dgmt.top` AAAA 记录 → workstation → Caddy → `forward_auth` 委托 auth.dgmt.top → 后端服务
3. **SSH 运维** → `lab.wineryz.top:2200X` → cloud → ssh-fwd systemd 服务 → Tailscale 内网设备 :22
4. **Tailscale 直连** → `100.64~99.X.X.X` → 设备 SSH/API（需加入 Tailnet）

### 6.4 关键网络特点

- **IPv6 公网直连**：workstation、macmini01~04、spark01~02、pc01 均有电信 IPv6 公网地址（`240e:b8f:1d22:e700::/64`），可被 IPv6 客户端直达，无需经过 cloud 转发。
- **双栈 DNS**：`erp`、`toiv` 同时配置 A（→cloud）和 AAAA（→workstation）记录，客户端根据协议栈自动选择最优路径。
- **Tailscale 兜底**：当电信 IPv6 前缀变化或路由异常时，Tailscale Mesh 仍可保证设备互通；cloud 的 SSH 跳板也通过 Tailscale 转发至内网设备。
- **DDNS 自动刷新**：workstation 的 `ddns-aliyun.timer` 每分钟检测 IPv6 前缀变化并更新阿里云 DNS AAAA 记录，确保域名始终指向有效地址。
